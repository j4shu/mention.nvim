# Research: mini.nvim module structure and API conventions

Issue: #2. Primary source: local checkout `~/git/mini.nvim/` (read-only). Line refs are against that checkout.
Goal: decide which conventions mention.nvim adopts (single-module plugin: append `@path` / `@path#L1-5` mentions to a collection scratch buffer; toggle/clear/copy actions).

## 1. File layout

### mini.nvim monorepo (per module)

- `lua/mini/<name>.lua` — entire module in one file (e.g. `lua/mini/trailspace.lua`).
- `doc/mini-<name>.txt` — generated vimdoc.
- `readmes/mini-<name>.md` — per-module README source.
- No `plugin/` directory for any module; activation is purely `require('mini.<name>').setup()`.

### Standalone split repos (mini.jump etc.)

Generated by `scripts/dual_sync.sh` (`sync_module`, lines 15-69; tracked file list lines 26-31; confirmed by `MAINTAINING.md:277`). Contents:

```
lua/mini/<name>.lua
doc/mini-<name>.txt
README.md        (readmes/mini-<name>.md moved to root, dual_sync.sh:47-48)
LICENSE
```

- No `plugin/` dir in standalone repos either. Nothing runs before `setup()`.
- Release model: `main` = dev, `stable` branch + annotated tags (`scripts/dual_release.sh:36-52`, `readmes/mini-jump.md:42-45`).

### Recommendation for mention.nvim

```
lua/mention.lua      (or lua/mention/init.lua if it outgrows one file)
doc/mention.txt
README.md
LICENSE
```

Single file, no `plugin/` dir, lazy-activated via `require('mention').setup()`. YAGNI: skip `plugin/` unless a command must exist pre-setup (mini's answer: it must not).

## 2. Module table + `setup(config)` pattern

Canonical minimal template: `lua/mini/trailspace.lua`. Skeleton:

```lua
local Mention = {}
local H = {}

Mention.setup = function(config)
  _G.Mention = Mention              -- export global (trailspace.lua:59)
  config = H.setup_config(config)   -- validate + merge (line 62)
  H.apply_config(config)            -- store, create mappings (line 65)
  H.create_autocommands(config)     -- (line 68)
  H.create_default_hl()             -- (line 71)
end
```

Fixed ordering across all modules: export global first, then validate, apply, wire behavior. README.md:194: setup "always creates a global Lua object with coherent camel-case name" (for mappings via `<Cmd>lua ...<CR>` and user introspection).

Config machinery (trailspace.lua:127-142):

```lua
Mention.config = { ... }                                -- public defaults on module table
H.default_config = vim.deepcopy(Mention.config)          -- snapshot after literal

H.setup_config = function(config)
  H.check_type('config', config, 'table', true)
  config = vim.tbl_deep_extend('force', vim.deepcopy(H.default_config), config or {})
  H.check_type('<field>', config.<field>, '<type>')      -- per-field validation
  return config
end

H.apply_config = function(config) Mention.config = config end
```

Merge order invariant: deepcopied defaults as base, user config wins (`'force'`).

Runtime config resolution + disabling (trailspace.lua:169-173, jump.lua:388-392; README.md:202-212):

```lua
H.get_config = function(config)
  return vim.tbl_deep_extend('force', Mention.config, vim.b.mention_config or {}, config or {})
end
H.is_disabled = function() return vim.g.mention_disable == true or vim.b.mention_disable == true end
```

Every public entry point starts with `if H.is_disabled() then return end` (bufremove.lua:87,99).

Adopt: all of the above verbatim (names `Mention`, `vim.b.mention_config`, `vim.g.mention_disable`). Buffer-local config is cheap and matches mini's contract.

## 3. The `H` helpers table

Everything non-public lives in a single file-local `H` table, in section order:

- `H.default_config` — deepcopy of defaults.
- `H.cache` — runtime state (jump.lua:307-316); for mention.nvim: collection `buf_id`, `win_id`.
- `H.ns_id` — namespaces if extmarks/highlights needed (hipatterns.lua:667).
- Settings: `H.setup_config`, `H.apply_config`, `H.check_type`.
- Behavior: `H.create_autocommands`, `H.create_default_hl`, `H.create_user_commands` (only if commands), `H.get_config`, `H.is_disabled`.
- Utilities: `H.error`, `H.notify`, `H.map`, buffer helpers.

Exact helper implementations to copy:

```lua
-- trailspace.lua:199 (level 0 strips file:line position)
H.error = function(msg) error('(mention) ' .. msg, 0) end

-- trailspace.lua:201-204
H.check_type = function(name, val, ref, allow_nil)
  if type(val) == ref or (ref == 'callable' and vim.is_callable(val)) or (allow_nil and val == nil) then return end
  H.error(string.format('`%s` should be %s, not %s', name, ref, type(val)))
end

-- git.lua:1707 (level passed as string key)
H.notify = function(msg, level_name) vim.notify('(mention) ' .. msg, vim.log.levels[level_name]) end

-- jump.lua:593-597 (empty lhs = disabled; silent by default)
H.map = function(mode, lhs, rhs, opts)
  if lhs == '' then return end
  opts = vim.tbl_deep_extend('force', { silent = true }, opts or {})
  vim.keymap.set(mode, lhs, rhs, opts)
end
```

Message prefix convention: `(mini.<name>) ` on all three channels (error, vim.notify, echo). For us: `(mention) `. Some modules add a `silent` third arg to `H.notify` (jump2d.lua:1089-1091); adopt only if a `config.silent` option is added.

## 4. Commands, mappings, autocommands, highlights

- **Mappings**: declared in `config.mappings` (each value one lhs string; `''` disables — comment convention at jump.lua:105). Applied once inside `H.apply_config` via `H.map`, rhs as `'<Cmd>lua Mention.toggle()<CR>'` (jump.lua:351-371). Changing `config.mappings` after setup has no effect (README.md:198). mini has no "no default mappings" dogma; ~19 modules ship defaults but every one is individually disableable via `''`.
  - Recommendation: `config.mappings = { add = '...', toggle = '...', clear = '', copy = '' }` with conservative or empty defaults; empty string disables.
- **User commands**: minority pattern; only colors/git/deps/pick/cmdline create them, via `H.create_user_commands` called from setup (git.lua:257,662-665). Recommendation: skip user commands initially (YAGNI); Lua API + mappings suffice. If added later, one `:Mention` command with subcommand completion, mini.git style.
- **Autocommands** (trailspace.lua:144-165): `H.create_autocommands` with augroup `Mention` cleared on re-setup, local `au(event, pattern, callback, desc)` closure:

```lua
local gr = vim.api.nvim_create_augroup('Mention', {})
local au = function(event, pattern, callback, desc)
  vim.api.nvim_create_autocmd(event, { group = gr, pattern = pattern, callback = callback, desc = desc })
end
au('ColorScheme', '*', H.create_default_hl, 'Ensure colors')
```

  The `ColorScheme -> H.create_default_hl` autocmd is standard (trailspace.lua:164) so highlights survive colorscheme changes.
- **Highlights** (trailspace.lua:167): `default = true` so user overrides win:

```lua
H.create_default_hl = function() vim.api.nvim_set_hl(0, 'MentionNormal', { default = true, link = 'NormalFloat' }) end
```

## 5. Scratch/special buffer management (for the collection buffer)

Cross-module conventions (deps.lua:1609, starter.lua:1467, notify.lua:897, files.lua:3069, pick.lua:3628):

- **Naming**: every module has `H.set_buf_name`:

```lua
H.set_buf_name = function(buf_id, name)
  vim.api.nvim_buf_set_name(buf_id, 'mention://' .. buf_id .. '/' .. name)
end
```

- **Creation**: `nvim_create_buf(false, true)` (unlisted scratch) for internal UI; `nvim_create_buf(true, true)` (listed scratch) for buffers the user should navigate to (deps confirm buffer deps.lua:1423, notify history notify.lua:542).
- **Options**: set via `vim.bo[buf_id]` (notify/files/pick). `scratch=true` in `nvim_create_buf` already gives `buftype=nofile`, `bufhidden=hide`, `noswapfile`. Set `filetype = 'mention'` (notify.lua:745 pattern) for user autocmd/ftplugin hooks.
- **Reuse on toggle**: cache ids in `H.cache` and gate on validity, mini.notify style (notify.lua:477-495, 600-608):

```lua
H.cache = { buf_id = nil, win_id = nil }
-- reuse: if not (buf_id and vim.api.nvim_buf_is_valid(buf_id)) then create end
```

  Toggle idiom from mini.files (files.lua:384-387): `close()` returns whether it closed anything; `toggle = if not close() then open() end`. Window validity via `nvim_win_is_valid` + tabpage check (notify.lua:483).
- **Content writes**: `nvim_buf_set_lines`; only toggle `modifiable` if buffer is kept `nomodifiable` (starter.lua:381-383). For mention.nvim the collection buffer should stay modifiable (user may hand-edit the mention list), so no toggling needed.
- **Alternative lookup**: mini.notify finds its history buffer by scanning `vim.bo[id].filetype == 'mininotify-history'` instead of caching (notify.lua:538-545) — resilient to plugin reload. Fine as fallback; primary should be `H.cache`.
- **Cleanup**: `pcall(vim.api.nvim_buf_delete, buf_id, { force = true })` (deps.lua:1430, starter.lua:404).
- mini.deps `acwrite` + `BufWriteCmd` pattern (deps.lua:1453-1472) is for save-triggers-action confirm buffers; not needed here.

## 6. Decision summary for mention.nvim scaffolding

Adopt:

1. Layout: `lua/mention.lua` single file, `doc/mention.txt`, `README.md`, no `plugin/` dir.
2. `Mention` module table + `H` local table; `setup()` order: `_G.Mention = Mention` -> `H.setup_config` -> `H.apply_config` -> `H.create_autocommands` -> `H.create_default_hl`.
3. Config: `Mention.config` defaults, `H.default_config` deepcopy, `tbl_deep_extend('force', ...)` merge, `H.check_type` validation, `H.get_config` with `vim.b.mention_config`, `H.is_disabled` with `vim.g/vim.b.mention_disable`.
4. `config.mappings` table, `''` disables, applied once in `H.apply_config` via `H.map` (rhs `<Cmd>lua Mention.x()<CR>`).
5. Messages: `(mention) ` prefix; `H.error(msg)` with level 0; `H.notify(msg, 'WARN')` via `vim.log.levels`.
6. Collection buffer: listed scratch `nvim_create_buf(true, true)`, name `mention://<bufnr>/collection`, `filetype = 'mention'`, cached in `H.cache.buf_id`/`win_id` with validity-gated reuse; toggle = `if not close() then open() end`.
7. Augroup `Mention` with `ColorScheme` re-apply of default highlights (only if/when highlights exist).

Skip (YAGNI): user commands, `plugin/` dir, `acwrite`/BufWriteCmd machinery, floating-window rendering, `config.silent` (until requested).
